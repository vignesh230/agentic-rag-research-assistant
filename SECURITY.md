# Security Policy

## Supported versions

Only the latest version on `main` receives security fixes.

## Reporting a vulnerability

Please **do not** open a public issue for security vulnerabilities.

Email **vigneshgovindu02@gmail.com** with:
- A description of the vulnerability
- Steps to reproduce
- Potential impact

You will receive a response within 72 hours. Once confirmed, a fix will be released as soon as practical and you will be credited in the release notes (unless you prefer to remain anonymous).
